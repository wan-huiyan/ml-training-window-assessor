XGBoost Sparsity-Aware Routing
Tree Node: split on logins_last_30d
│ NaN (pre-Mixpanel) → CRM-only subtree (account_tier, contract_length, NPS...)
│ Value (post-Mixpanel) → CRM + usage subtree (all features available)
Per-node, per-feature routing — Algorithm 2, Chen & Guestrin 2016
Critical Pitfall
⚠ .fillna(0) destroys XGBoost's missing value handling
Current pipeline: .fillna(0).astype(int) for product usage event counts.
This makes “no tracking existed” (structural NaN) indistinguishable from “tracked but zero usage” (genuine 0).
Fix: Gate .fillna(0) on target_date >= 2024-11-01. Pre-Mixpanel rows stay NaN.
